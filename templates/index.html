/genes

Get 127.0.0.1::5002/genes

Accepts Parameters:
  • lookup (string, required): must have a minimum of three characters
  • species (string, optional)
 Sample request http://127.0.0.1:5002/genes?lookup=aba&species=taeniopygia_guttata
Response 
{
  "data": [
    {
      "display_label": "ABAT",
      "location": "14:5274801-5300346",
      "species": "taeniopygia_guttata",
      "stable_id": "ENSTGUG00000004611"
    }
  ]
}

Unit Testing?
Unit Testing! In my current company we have been using PHPUnit, which has proven valid. For an API service, you would need to test against valid inputs to ensure they are valid. For this particular example, I would be able to check the "wild-card" nature of the lookup against a small list of expected results. A Library like Nose would be able to mock and assert responses to check whatever I am passing it.
